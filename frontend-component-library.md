# Frontend Component Library

> Build domain specific, psychologically (behavior + cognitively) aligned components that enhance accessibility and useability of components hand tailored for the industries that they are being created for.

**Category**: Developer Tools
**Tags**: frontend, component, library, accessibility, usability, color theory, ux, behavioral standards

## Purpose
Build a consistent backbone for applications requiring frontend components.

## Instructions
1. Design and develop frontend components that are accessible, usable, and tailored to specific industries.
2. Consider color theory, UX behavioral standards, and cognitive psychology when creating components.

## Prompt Template
```
Develop a {{type}} {{industry}} {{feature}} that is {{adjective}} and meets {{criterion}}.
```

## Examples
### Accessible Button
**Input**

```
Design a button that is accessible for users with visual impairments.
```

**Output**

```
A button with high contrast colors and clear typography.
```

### Industry-Specific Navigation
**Input**

```
Create a navigation menu for a healthcare application.
```

**Output**

```
A navigation menu with clear labels and icons that are relevant to the healthcare industry.
```

## Tests
### Accessible Button Test
**Input**

```
A button with low contrast colors and unclear typography.
```

**Expected**

```
The button is not accessible for users with visual impairments.
```

### Industry-Specific Navigation Test
**Input**

```
A navigation menu with generic labels and icons.
```

**Expected**

```
The navigation menu is not relevant to the healthcare industry.
```
